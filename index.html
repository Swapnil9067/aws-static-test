Deployment Steps

1
Create S3 Bucket
Create an S3 bucket with a unique name, disable Block all public access, and enable static website hosting with index.html as the index document.
# AWS CLI command aws s3 mb s3://swapnil-static-website --region ap-south-1 aws s3 website s3://swapnil-static-website \ --index-document index.html
2
Attach Bucket Policy (Public Read)
Attach a bucket policy to allow public read access to all objects. This makes the website accessible to anyone with the URL.
# bucket-policy.json { "Version": "2012-10-17", "Statement": [{ "Effect": "Allow", "Principal": "*", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::swapnil-static-website/*" }] }
3
Create IAM User (Least Privilege)
Create a dedicated IAM user for GitHub Actions with only the minimum permissions needed — PutObject and DeleteObject on this specific bucket only.
# iam-deploy-policy.json { "Version": "2012-10-17", "Statement": [{ "Effect": "Allow", "Action": [ "s3:PutObject", "s3:DeleteObject" ], "Resource": "arn:aws:s3:::swapnil-static-website/*" }] }
4
Set Up GitHub Actions CI/CD
Create the workflow file in your repo. Add AWS credentials as GitHub Secrets. On every push to main, the pipeline auto-deploys to S3.
# .github/workflows/deploy.yml name: Deploy to S3 on: push: branches: [main] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 - name: Configure AWS Credentials uses: aws-actions/configure-aws-credentials@v2 with: aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }} aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }} aws-region: ap-south-1 - run: aws s3 sync . s3://swapnil-static-website --delete
5
Set Up CloudFront Distribution
Create a CloudFront distribution pointing to your S3 bucket. This adds HTTPS, global CDN caching, and faster load times worldwide.
# Key CloudFront settings Origin domain: swapnil-static-website.s3-website.ap-south-1.amazonaws.com Viewer protocol: Redirect HTTP to HTTPS Cache policy: CachingOptimized Price class: Use only Asia and India edge locations
